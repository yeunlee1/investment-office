# 투자 분석 사무실

미국과 한국 주식 후보를 같은 기준시점의 가격·거시·재무·공시 자료로 분석하고, 여섯 역할의 의견과 결정론적 위험 정책을 한 장의 결정 카드로 정리하는 로컬 애플리케이션입니다. 사용자가 승인·보류·기각을 직접 기록해야 하며 증권사 주문을 만들거나 전송하지 않습니다.

## 화면 구성

- 운영 허브는 최근 개별 분석, 종목추천, 예약 작업과 완료 이력을 요약합니다.
- 시장 관제실은 공통 거시 기준선과 미국·한국 시장 국면, 거시 품질, 공급원 준비 상태를 표시합니다.
- 개별 종목 화면은 미국 티커 또는 한국 여섯 자리 종목코드를 접수하고 에이전트 진행 상황을 보여줍니다.
- 종목추천 화면은 시장별 대표 유니버스를 먼저 선별한 뒤 선택한 후보만 심층 분석합니다.
- 완료 이력은 과거 결정 카드, 업무, 회의와 사람의 검토 결과를 다시 보여줍니다.
- 보류 중인 2차원 사무실은 `/office`에 그대로 남아 있지만 현재 개발 우선순위에서는 제외되어 있습니다.

## 분석 흐름

1. 시장별 완료 일봉을 수집하고 기술 지표를 계산합니다.
2. 공통 거시와 한국 고유 거시, 공식 재무·공시를 병렬 수집합니다.
3. 모든 값에 출처, 관측 시각, 공개 시각, 수집 시각과 기준시점을 붙여 연구 번들을 만듭니다.
4. 필수 구역의 공백·노후·식별자 충돌을 검사해 분석 가능 여부를 확정합니다.
5. 기본면·기술·뉴스 역할, 상승·하락 토론과 위원장이 같은 사실 원장을 사용합니다.
6. 결정론적 위험 정책이 기술 위험과 시장 국면을 반영해 비중 상한을 계산합니다.
7. 자료 품질이 불충분하면 긍정적인 에이전트 의견과 무관하게 신규 포지션을 차단합니다.
8. 사용자가 최종 결정을 검토하고 기록합니다.

에이전트 호출은 로컬 코덱스 명령줄 도구에 저장된 정액제 로그인을 사용합니다. 각 호출은 임시 작업공간과 읽기 전용 격리로 실행되며 외부 도구와 웹 검색을 사용할 수 없습니다. 에이전트는 입력 사실 식별자만 선택하고 사람이 보는 근거 값·단위·출처·공개시각은 서버가 사실 원장에서 결정론적으로 다시 만듭니다.

## 데이터 공급 순서

| 범위 | 우선 공급원 | 대체 또는 보조 | 적용 원칙 |
|---|---|---|---|
| 미국 완료 일봉 | 티잉고 조정 일봉 | 없음 | 계약 토큰이 없거나 공급자 장애가 나면 사용 불가로 차단합니다. |
| 한국 완료 일봉 | 공공데이터포털 금융위원회 주식시세 | 없음 | 다음 영업일 공식 자료와 비조정주가 한계를 표시하며 비공식 대체는 하지 않습니다. |
| 공통 거시 | 미국 재무부·미국 노동통계국 | 연준 이사회·에너지정보청 어댑터 예정 | 현재 금리와 성장·물가를 수집하고 권리 확인 전인 달러·변동성·원유·유동성 축은 필수 차단합니다. |
| 한국 고유 거시 | 한국은행 경제통계시스템 | 없음 | 기준금리, 국고채 3년·10년, 원·달러, 소비자물가를 사용합니다. |
| 미국 재무·공시 | 미국 증권거래위원회 | 없음 | 회사 사실과 최근 연차·분기·수시 공시 메타데이터만 사용합니다. |
| 한국 재무·공시 | 금융감독원 전자공시 | 없음 | 연결 재무제표와 최근 공시 메타데이터를 사용합니다. |
| 독립 언론 뉴스 | 아직 분석 입력에 미연결 | 빅카인즈·네이버는 탐색 전용 | 전문 이용 권한이 확인되지 않은 본문은 분석 근거로 저장하지 않으며 신규 포지션을 차단합니다. |

공급원 정책과 현재 인증 준비 상태는 시장 관제실과 `/api/data-sources`에서 확인할 수 있습니다. FRED·야후·라이선스 확인 전인 로이터 커넥트는 기본 비활성 상태입니다.

## 실행 설정

`.env.example`을 참고해 필요한 값만 로컬 `.env`에 추가합니다. 비밀값은 저장소에 커밋하지 않습니다.

```text
INVESTMENT_OFFICE_SEC_USER_AGENT=
INVESTMENT_OFFICE_TIINGO_API_TOKEN=
INVESTMENT_OFFICE_DATA_GO_KR_SERVICE_KEY=
INVESTMENT_OFFICE_DART_API_KEY=
INVESTMENT_OFFICE_BOK_ECOS_API_KEY=
```

- 미국 증권거래위원회 사용자 식별자는 응용프로그램 이름과 연락 가능한 정보를 포함해야 합니다.
- 키가 없는 공급원은 조용히 빈 값으로 대체되지 않고 사용 불가 구역과 차단 사유를 만듭니다.
- 계약 또는 공식 가격 공급원이 준비되지 않으면 비공식 대체 없이 분석을 차단합니다.

## 처음 한 번 실행

윈도우 파워셸에서 실행합니다.

```powershell
uv sync
.\scripts\install_mariadb.ps1
uv run python .\scripts\bootstrap_database.py
.\scripts\start.ps1
```

데이터베이스 설치 스크립트는 프로젝트 전용 서버를 `127.0.0.1:3307`에서 실행합니다. 기존 데이터베이스 구조를 변경하는 작업은 별도 승인 없이 실행하지 않습니다.

## 접속과 종료

- 운영 허브는 `http://127.0.0.1:8765/`입니다.
- 시장 관제실은 `http://127.0.0.1:8765/markets`입니다.
- 개별 분석은 `http://127.0.0.1:8765/analysis`입니다.
- 종목추천은 `http://127.0.0.1:8765/discovery`입니다.
- 완료 이력은 `http://127.0.0.1:8765/history`입니다.
- 웹 서버 종료 명령은 `.\scripts\stop_app.ps1`입니다.

## 데이터베이스 범위

기존 `candidates`, `analysis_runs`, `agent_outputs`, `events`, `reviews`, `snapshots` 테이블만 사용합니다. 시장, 출처, 사실 원장과 품질 보고서는 기존 제이슨 필드와 스냅샷에 저장하므로 이번 개편에는 스키마 변경이나 마이그레이션이 없습니다.

## 투자 사용 범위

결정 카드는 사람의 판단을 돕는 분석 초안입니다. 세금, 수수료, 슬리피지, 개인 보유 종목, 유동성 제약과 모든 뉴스 사건을 완전하게 반영하지 못합니다. 사용자는 주문 전에 가격과 원문 공시를 다시 확인하고 허용 손실을 직접 정해야 합니다.

상세 설계와 자료 품질 규칙은 [시장 연구 설계](docs/시장_연구_설계.md)에 정리되어 있습니다.
